利用hexo创建博客--折腾与踩坑记录：2

北京

这篇文章就讲一下关于文章发布和评论系统部署、插件安装的事。 

阅读须知：默认已经部署完博客且使用Acrylic-Next主题

文件

上篇结尾处成功上传博客到GitHub服务器中部署，访问用户名.github.io后，界面大概是这样：

接下来讲一下问价夹中内容。

hexo文件夹
source

这里存放着写博文需要用到的文件。
路径为E:\workplace\workplace\source\_posts文件夹是存放博客文章的。使用hexo new新建文章，或者安装hexo-admin插件，后文会讲。

其中.md文件就是你的博文，使用记事本打开就可以编辑，或者用别的，只要是用MarkDown语法即可。
MarkDown教程
source文件夹下新建文件夹以存放博文中图片，引用时路径/<文件夹名称>/<图片名称>.png即可。 

这里有可能会出现一个小问题，即本地写文章时图片能够加载但上传到GitHub就加载不出来，可安装hexo-renderer-marked插件解决，稍后会说。

draft文件夹为草稿，没有太大用途，除非是将博客部署在了自己电脑上再通过公网ip访问，一般不用管。
通常来讲，这就够了，接下来将讲解一下启用主题的 即刻 后存在的文件夹_data与says
通过将主题配置文件中bbtime选项改为true来启用即刻，根目录终端运行hexo new page says新建即刻。
_data中存放的是即刻的内容，可通过上篇文章中主题教程进行书写。says中存放的是即刻的配置文件index.ymd：

title: 即刻    #即刻标题
date: 2023-07-26 15:56:45    #创建时间
comments: true    #是否允许评论
aside: false    #侧边栏
type: says    #类型，需要手动添加

将写完的即刻文扔到_data即可。

theme

文件夹中存放了下载下来的主题文件。
如，这是我的：

主题文件夹

以Acrylic-Next为例。

source

其中最常用到的是img文件夹。（其余的什么用自己去改一改源码就知道了）
其中，icon存放的是主页中这个位置↓中的图标。

在启用主题配置banner选项后更改下方图片路径即可修改显示图标。
theme文件夹，存放的是一些无关紧要的小地方的图标，类似：

这些文件夹仅仅是分类方便，具体设置还是看主题配置文件中的路径

插件

上文中所说hexo-admin插件提供了hexo文章的可视化编写，运行npm install --save hexo-admin安装插件，输入localhost:4000/admin/即可。
具体教程详见https://blog.csdn.net/nineya_com/article/details/103380243 

 题外话： 如果说你的博客是运行在本地上的，那么建议设置密码防止他人直接访问管理界面修改文章，如果你的博客部署在了GitHub服务器上，那么放心，GitHub只支持静态网页，admin插件是不会一起传上去的，设置密码就没什么必要了。

再说hexo-renderer-marked，博客根目录运行npm install hexo-renderer-marked以安装插件，修改全局配置post_asset_folder为true即可，有时新建文章后并不会有同名文件夹出现，在\source\_posts文件夹中新建一个同名文件夹即可，引用时使用![图像说明](image.jpg)即可。

要说的大概也就这么多了，至于免费域名的注册，网上相关教程很多，不再过多赘述。其他相关内容我想起来的话会在后续发布，至此，本教程完结。
